## 常见问题排查

### 问题 1：客户端无法连接服务端

**可能原因**：
- WebSocket 地址配置错误
- 服务端未启动或崩溃
- 网络连接问题（防火墙、端口、云服务安全组）

**排查步骤**：
1. 检查客户端 config.js 中的 wsUrl 是否正确
2. 检查服务端 PM2 服务状态：`pm2 status`
3. 检查服务端日志：`pm2 logs print-admin`
4. 检查服务器防火墙是否开放 3000 端口
5. 使用 telnet 测试端口：`telnet 39.104.59.201 3000`

---

### 问题 2：打印机注册后没有标签

**可能原因**：
- 数据库中打印机记录的 tags 字段为空
- 打印机是新增的（默认标签未生效）

**排查步骤**：
1. 查看数据库：`SELECT id, name, tags FROM printers WHERE name = '打印机名称';`
2. 如果 tags 为空，手动设置：
   ```sql
   UPDATE printers SET tags = 'normal' WHERE id = X;
   ```
3. 重启客户端，服务端重新读取配置

---

### 问题 3：订单没有被分配打印机

**可能原因**：
- 没有符合条件的打印机（在线、空闲、启用、标签匹配）
- 订单标签配置错误
- 打印机状态同步延迟

**排查步骤**：
1. 查看订单信息：`SELECT * FROM orders WHERE id = X;`（确认 status 和 tags）
2. 调用可用打印机接口：`GET /api/printers/available/:tag`
3. 查看打印机和客户端日志
4. 查看打印机状态：`SELECT id, name, status, enabled, tags FROM printers WHERE id = X;`

---

### 问题 4：客户端收到打印任务但不执行

**可能原因**：
- 打印机名称不匹配
- PDF 文件下载失败
- 打印命令执行失败

**排查步骤**：
1. 查看客户端日志，确认任务详情
2. 检查打印机名称是否在 ALLOWED_PRINTERS 列表中
3. 检查 PDF 下载地址是否可访问
4. 检查打印工具是否安装（SumatraPDF）
