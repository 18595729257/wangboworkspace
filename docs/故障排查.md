# 故障排查

## 客户端无法连接

```bash
# 1. 检查 WebSocket 地址
curl -v https://xinbingcloudprint.top/ws/printer

# 2. 检查服务端状态
curl https://xinbingcloudprint.top/health

# 3. 查看服务端日志
ssh root@39.104.59.201 "pm2 logs print-admin --lines 30"
```

## 打印机注册后无标签

```sql
-- 手动设置标签
UPDATE printers SET tags = 'normal' WHERE id = <ID>;
-- 重启客户端生效
```

## 订单无打印机分配

1. 确认有打印机在线且空闲：`GET /api/printers`
2. 确认订单 `print_tag` 与打印机 `tags` 匹配
3. 确认打印机 `enabled = 1`
4. 查看服务端日志：`[分配]` 开头的日志

## 服务端启动失败

```bash
# 检查端口占用
ss -tlnp | grep 3000

# 检查 .env 配置
ssh root@39.104.59.201 "cat /opt/print-miniprogram-admin/.env | grep DB"

# 手动测试数据库连接
ssh root@39.104.59.201 "mysql -uroot print_admin -e 'SELECT 1'"
```

## Redis 未连接（警告）

Redis 为可选组件，未配置时缓存功能静默跳过，不影响核心功能。如需启用，在 `.env` 中添加：

```bash
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
```

## 小程序接口返回 400

检查请求的 `content-type` 是否正确，上传接口需要 `multipart/form-data`。

## WebSocket 502 Bad Gateway

Nginx 需要支持 WebSocket 升级，确认配置中有：

```nginx
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```
